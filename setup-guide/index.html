Windows下安装配置与连接

  1. 根据您的系统选择下载：openvpn-2.3-eduVPN-i686.exe.zip（32位）或openvpn-2.3-eduVPN-x86_64.exe.zip（64位）并安装。
  2. 修改您 本地连接 - Internet协议版本4（TCP/IPV4） 中DNS服务器为 8.8.8.8（首选DNS）8.8.4.4（备用DNS），如果您的学校提供了IPV6 DNS/或者您使用了一个openwrt路由器，您可以还需要将Internet协议版本6中的DNS设置为Google公共DNS或者设置为任何一个不可用的地址，否则会因为IPV6 DNS优先级更高而导致DNS污染。
  3. 新建一个空白文档，重命名为eduvpn.txt，将我们发送给您的用户名，密码保存到其中，譬如您的用户名为 asdf，密码为1234，则文本文档内容为：
    asdf
    1234
    然后将该文档复制到openvpn安装目录下的config文件夹，譬如（默认位置）：C:\Program Files (x86)\OpenVPN\config 。
  4. 您现在可以双击桌面的eduVPN，选择合适的配置文件连接了。
    5. 注意: 对于Vista，Windows 7 或更高版本的Windows系统，您必须点击右键，选择以管理员身份运行，否则openvpn将没有权限设置路由。导致您虽然连接成功却不能访问网络。

Mac OSX下安装配置与连接

  1. 下载TunnelBlick 3.0 final with openvpn IPv6 patch并安装。
  2. 请点击下载 config.zip，找到找到里面的所有文件。并新建包含您用户名密码的eduvpn.txt（请参考Windows下建立eduvpn.txt的步骤）。将所有文件放到 ~/Library/Application Support/Tunnelblick/Configurations （注意是用户目录下的Library目录，而非根目录下的Library目录）目录下面。
  3. 现在您可以选择合适的配置文件连接了。

Linux 下安装配置与连接

    拥有完整IPV6支持的2.3版本已经正式发布了，经过我们的测试没有问题。推荐使用，如果发行版提供的openvpn提示错误不能从文件中读取密码，则需要自己编译并加上--enable-password-save

  1. 首先是解决依赖问题。编译并使用openvpn需要如下依赖：gcc，lzo-devel，openssl-devel，make，patch，libgcrypt-devel等。您需要使用自己发行版的包管理工具安装上这些依赖。 对于centos，fedora等红帽系系统，请使用yum安装： 
    yum install patch gcc lzo-devel openssl-devel make libgcrypt-devel
    对于Debian，ubuntu等debian系系统，请使用apt-get安装： 
    apt-get install patch gcc lzo-devel openssl-devel make libgcrypt-devel
  2. 获取openvpn源码，从2.3开始我们已经不需要额外的patch了： 
    wget http://swupdate.openvpn.org/community/releases/openvpn-2.3.2.tar.gz
    tar -zxvf openvpn-2.3.2.tar.gz
  3. 编译安装。我们启用--enable-password-save选项从而可以从文件读取用户名和密码。
    ./configure --enable-password-save 
    make 
    sudo make install
  4. 请点击下载 config.zip，找到里面的所有文件。并新建包含您用户名密码的eduvpn.txt（请参考Windows下建立eduvpn.txt的步骤）。
    建立配置文件文件夹，并放置所有文件到其中
    sudo mkdir /etc/openvpn 
    sudo cp xxx /etc/openvpn
    连接
    sudo openvpn --cd /etc/openvpn --config xxx_xxx.ovpn --daemon
  5. 您也可以参考源代码目录sample\sample-config-files\下的 openvpn-startup.sh ，使用rc.local 而使openvpn可以开机自动运行。请直接阅读文件内容完成设置。

移动终端的使用

虽然现在iOS（iphone，ipad）以及Android都已经有官方的openvpn支持了，我们还是强烈建议在上述的两个终端使用pptp或者shadowsocks这两个更轻量的替代。

如果您一定要在iOS或者Android上使用openVPN，请直接在相应的市场安装openVPN官方的客户端，根据客户端指示操作即可。如果有什么问题欢迎提交ticket咨询。

这里提供了除默认路由方式以外的openVPN配置文件。

这些配置文件有什么不同：

这些配置文件包含了不同的路由表。通过让特定的IP不通过VPN服务器而达到下面这些目的：

  • 已经有了国内的访问权限，让国内地址不通过路由来获取更好的访问速度
  • 需要在使用VPN的时候访问其他校内地址，譬如校内的论坛，教学服务中心，校内BT等

如何选择配置文件：

如果没有特殊需求的话，位于日本的Sakura服务器应该能满足您的所有需求，请选择对应您学校的Sakura服务器配置文件即可。
不过如果您需要访问一些只能由美国IP访问的站点或者Sakura不可用可以选择HE或者Blue。

获取配置文件以后：

  • Windows用户请将配置文件放置到安装目录下面的config目录
  • Linux用户请放置到您指定的配置文件目录，一般为 /etc/openvpn
  • Mac OS用户请放置到 ~/Library/Application Support/Tunnelblick/Configurations
