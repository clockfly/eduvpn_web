安装配置

Windows下安装配置与连接

  1. 下载openvpn-2.3-eduVPN-i686.exe.zip（32位）或openvpn-2.3-eduVPN-x86_64.exe.zip（64位）并安装。
  2. 修改您 本地连接 - Internet协议版本4（TCP/IPV4） 中DNS服务器为 8.8.8.8（首选DNS）8.8.4.4（备用DNS）。
  3. 新建一个空白文档，重命名为eduvpn.txt，将我们发送给您的用户名，密码保存到其中，譬如您的用户名为 asdf，密码为1234，则文本文档内容为：
    asdf
    1234
    然后将该文档复制到openvpn安装目录下的config文件夹，譬如（默认位置）：C:\Program Files (x86)\OpenVPN\config 。
  4. 您现在可以双击桌面的eduVPN，选择合适的配置文件连接了。
    注意：对于Vista，Windows 7 或更高版本的Windows系统，您必须点击右键，选择以管理员身份运行，否则openvpn将没有权限设置路由。导致您虽然连接成功却不能访问网络。

Mac OSX下安装配置与连接

  1. 下载TunnelBlick 3.0 final with openvpn IPv6 patch并安装。
  2. 请点击下载 config.zip，找到找到里面的所有文件。并新建包含您用户名密码的eduvpn.txt（请参考Windows下建立eduvpn.txt的步骤）。将所有文件放到 ~/Library/Application Support/Tunnelblick/Configurations （注意是用户目录下的Library目录，而非根目录下的Library目录） 目录下面。
  3. 现在您可以选择合适的配置文件连接了。

Linux 下安装配置与连接

    因为目前大部分发行版中的openvpn尚未启用ipv6支持，且编译时默认没有启用password-save。所以我们推荐所有的Linux用户自行编译openvpn。因为2.3还处在alpha阶段，所以我们使用稳定的2.2.0版搭配jjo的补丁。

  1. 首先是解决依赖问题。编译并使用openvpn需要如下依赖：gcc，lzo-devel，openssl-devel，make，patch，libgcrypt-devel等。您需要使用自己发行版的包管理工具安装上这些依赖。 
    对于centos，fedora等红帽系系统，请使用yum安装：
yum install patch gcc lzo-devel openssl-devel make libgcrypt-devel
对于Debian，ubuntu等debian系系统，请使用apt-get安装：
apt-get install patch gcc lzo-devel openssl-devel make libgcrypt-devel
  2. 获取openvpn和jjo ipv6 patch源代码。并应用jjo的ipv6 patch。 
    wget http://swupdate.openvpn.org/community/releases/openvpn-2.2.0.tar.gz
wget https://github.com/downloads/jjo/openvpn-ipv6/openvpn-2.2.0-ipv6-0.4.16.patch.gz
tar -zxvf openvpn-2.2.0.tar.gz 
gzip -d openvpn-xxx-ipv6-xxx.patch.gz 
patch -p0 < openvpn-xxx-ipv6-xxx.patch
  3. 编译安装。我们启用--enable-password-save选项从而可以从文件读取用户名和密码。 
    ./configure --enable-password-save
make 
sudo make install
  4. 请点击下载 config.zip，找到里面的所有文件。并新建包含您用户名密码的eduvpn.txt（请参考Windows下建立eduvpn.txt的步骤）。
    建立配置文件文件夹，并放置所有文件到其中
    sudo mkdir /etc/openvpn
sudo cp xxx /etc/openvpn
    连接
    sudo openvpn --cd /etc/openvpn --config xxx_xxx.ovpn --daemon
  5. 您也可以使用源码目录下的init.d脚本：sample-scripts/openvpn.init 而使openvpn可以开机自动运行。请直接阅读文件内容完成设置。
进阶配置

  1. 蹭移动，电信，联通热点或者假期在家短期使用IPV4的VPN

    2. 电信，移动以及联通在大城市都覆盖有热点但是需要通过相应的认证后才能使用，但是在连接上热点/网络以后，未对DNS查询（53端口流量）做限制。您可以在连上相应热点以后，不通过认证，直接使用我们给的配置文件中的eduvpn-hk-4.ovpn，eduvpn-jp-4.ovpn进行连接。然后就可以正常使用网络了。

    当然您暑假在家要短期使用ipv4 VPN也可以连接上述配置文件。

  2. 设定特定IP/IP段 不通过VPN直接连接

    3. 这在使用某些校内服务的时候特别有用（比如使用学校IP认证的服务，或者校内的BT服务，图书馆服务等。）。以下叙述稍显复杂，如果您有需要又不想自己动手，请在客户中心提交支持单，并注明您所在的学校，我们将在一个工作日内为您做好路由表
    首先您需要找到您的学校分配到的IP段，您需要用到：纯真IP数据库并使用APNIC提供的IP WHOIS工具验证。
    您获取到的IP数据经过整理以后应该是像下面这样（以北师大为例）：

    	59.64.32.0      59.64.63.255    北京市 北京师范大学
	114.255.218.0   114.255.218.255 北京市 北京师范大学
	202.112.80.0    202.112.95.255  北京师范大学 艺术楼机房
	210.31.64.0     210.31.79.255   北京师范大学  CZ88.NET
	219.142.99.0    219.142.99.31  北京市 北京师范大学
	219.142.121.0   219.142.121.63 北京市 北京师范大学
	219.224.16.0     219.224.31.255  北京市 北京师范大学
    然后转化为openvpn使用的路由格式：
    	route 59.64.32.0 255.255.224.0 net_gateway 
	route 114.255.218.0 255.255.255.0 net_gateway 
	route 202.112.80.0 255.255.240.0 net_gateway 
	route 210.31.64.0 255.255.240.0 net_gateway 
	route 219.142.99.0 255.255.255.224 net_gateway 
	route 219.142.121.0 255.255.255.192 net_gateway 
	route 219.224.16.0 255.255.240.0 net_gateway
    请注意观察将IP块转化为掩码格式的规律。转换完成后添加到配置文件最后即可。
